# replit.md

## Overview

This is a full-stack TypeScript web application using React on the frontend and Express on the backend. The project follows a monorepo structure with clear separation between client, server, and shared code. It appears to be a starter template or boilerplate designed for building modern web applications with a comprehensive UI component library based on shadcn/ui.

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### Project Structure
- **client/** - React frontend application with Vite as the build tool
- **server/** - Express.js backend API server
- **shared/** - Shared TypeScript types and database schema used by both client and server
- **migrations/** - Database migration files generated by Drizzle Kit

### Frontend Architecture
- **Framework**: React with TypeScript
- **Build Tool**: Vite with hot module replacement (HMR)
- **Routing**: Wouter (lightweight React router)
- **State Management**: TanStack React Query for server state
- **UI Components**: shadcn/ui components built on Radix UI primitives
- **Styling**: Tailwind CSS with CSS variables for theming (supports light/dark modes)
- **Form Handling**: React Hook Form with Zod validation via @hookform/resolvers
- **Internationalization**: JSON-based i18n with English and Spanish support

### Backend Architecture
- **Framework**: Express.js 5.x
- **Language**: TypeScript with tsx for development execution
- **API Pattern**: RESTful endpoints prefixed with `/api`
- **Storage Interface**: Abstracted storage layer (`IStorage`) with in-memory implementation (`MemStorage`) - designed to be swapped for database storage

### Database Layer
- **ORM**: Drizzle ORM with PostgreSQL dialect
- **Schema Location**: `shared/schema.ts` - contains table definitions and Zod validation schemas
- **Migrations**: Managed via Drizzle Kit (`drizzle-kit push` for development)
- **Schema Validation**: drizzle-zod for generating Zod schemas from Drizzle tables

### Build System
- **Development**: Vite dev server proxied through Express for unified development experience
- **Production Build**: 
  - Client: Vite builds to `dist/public`
  - Server: esbuild bundles to `dist/index.cjs` with selective dependency bundling for faster cold starts
- **Type Checking**: TypeScript with strict mode enabled

### Path Aliases
The project uses extensive path aliases configured in both TypeScript and Vite:
- `@/*` → `client/src/*`
- `@shared/*` → `shared/*`
- `@assets` → `attached_assets/`

## External Dependencies

### Database
- **PostgreSQL**: Primary database (configured via `DATABASE_URL` environment variable)
- **connect-pg-simple**: PostgreSQL session store for Express sessions

### UI Library
- **GC-UI-COMPONENTS**: External UI component library providing layout components (SidebarLayout, etc.)
- **Radix UI**: Complete suite of accessible, unstyled UI primitives
- **Lucide React**: Icon library

### Third-Party Services (dependencies present, implementation pending)

- **OpenAI**: AI/LLM integration
